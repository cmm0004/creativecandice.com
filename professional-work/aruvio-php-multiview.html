Page-Specific JS

Javascipts that only appear on certian pages, even though the pages all use the same template.

Because aruvio.com is a wordpress site, and all of the product pages use the same template file, adding a tracking JS on just a specific set of them (landing pages) provides an interesting challenge. Luckily, wordpress's documentation provides for such a need with the is_page() function, which lets you pass a page title and it returns True if the page title you asked for is the one currently being displayed.

This goes in the active theme's function.php file:

 /** this function is so tracking javascript 
 only appears on the pages we need it to.
	
	**/
function candice_get_pixel() {

		
	
	if( is_page( 'policy-management-software' ) ){
		print '[tracking script for policy page]';
	
	}elseif( is_page( 'compliance-management') ){
		print  '[tracking script for compliance page]';
	
	}elseif( is_page( 'third-party-management-application') ){
		print '[tracking script for third party risk page]';
	
	}elseif( is_page( 'aruvio-controls') ){
		print '[tracking script for controls page]';
	
	}else {
	return;
	}
}

And this goes in the functions.php file of our theme, as part of the wp_footer() action that delivers all the javascript to the footer template, before the closing <body> tag:

<?php 
function candice_print_javascript(){ ?>
[ all of our other javascript, 
google analytics, 
the live chat box, 
etc  ]

<?php 
candice_get_pixel() ?>
<?php } ;

add_action('wp_footer', candice_print_javascript'); ?>

I like to write my customizations as 'candice_...' just to keep it clear what I've added and what was already there. That way I can cntr-f 'candice' and see what I've changed.
