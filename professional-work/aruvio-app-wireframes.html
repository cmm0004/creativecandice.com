Mobile App Wireframes

Creating an app to demonstrate features of the Aruvio GRC product

I created a set of wireframes to guide the design of an Aruvio mobile app using the Balsamiq mockup tool. The app will be used as a presentation tool by our sales team, to use when they are out and about.

I am full vertical on this project, from design to deployment.

Flow Planning

I started by making a tree chart to plan where I wanted each of the screens to show, and the depth of the navigation. The plan I have is never more than two levels deep. In a future version of this app, I'd like to include a pricing calculator for Enterprise pricing, but right now, what I want is a minimal viable product. Changes will be iterative and incremental as I discuss with the sales team things they want and to add to later versions.

Startscreen

The app will open up on the 'Applications' screen with a short description and menu to switch to the 'Solutions' and 'Pricing' page.

In the content area of the App screen will be buttons that will take the user to individual Application description screens.

The 'Solutions' screen will follow the same paradigm as the 'Application' screen and the 'Pricing' screen will have an accordion that echoes the information found on the website here.

Individual Applications and Solutions screens

Each individual application or solution will have a set of three screens that the user will swipe right to left. A set of dots at the bottom of the screen provides a sort of map to let the user know where they are inside the screen set.

The first screen is a description of the product, the second has screenshots of the product in action, and the third is an easily digestible set of benefits for using the product.
