Aruvio Query Strings

A single 'thank you page' that populates with a different download link depending on how you got to the 'thank you page'.

View Live Site

This was an end-user experience problem, and an admin-user experience problem. We are using a wordpress installation for the aruvio website, which allows us to edit our content easily and quickly, but at the cost of being unable to make structural changes without some serious surgery. I needed (within the framework we already had in place) a way to have one thank you page that populated with different links depending on which downloadable was asked for through our web forms.

Well, it turns out our web-forms plugin allows you to pass query strings when you submit a form and then redirect the user to a new page, so I just set up another form on the thank you page with hidden HTML fields that only show when certain values are passed via query string in the url. These fields have links to the pdf the user requested.

This solution also immensely simplified our conversion analytics, since any download from the site is considered a 'conversion'.

A example using our Regulatory Compliance downloadable

I also used a similar set-up for the 'Quick Apply' section of our Careers page on the parent company, virima.com site.
